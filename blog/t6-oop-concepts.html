Blocs, Procs, and Lambdas

---Week 6

Blocs, Procs, and Lambdas are closures - a closure is a function or reference to a function together with a referencing environment—a table storing a reference to each of the non-local variables (also called free variables or upvalues) of that function.[1] A closure—unlike a plain function pointer—enables a function to access those non-local variables even when invoked outside its immediate lexical scope. In essence, a closure is a ways of grouping code we want to run.

The differences between Blocs and Procs are:

  • Procs are objects, blocks are not.
  • At most one block can appear in an argument list. In contrast, you can pass multiple procs to methods.

Although both Procs and Lambdas are both Proc objects, there are many differences, such as:

  • Lambdas check the number of arguments, while procs do not.
  • Lambdas and procs treat the ‘return’ keyword differently.
