Active Record

---Week 8

ActiveRecord can't be discussed without discussing MVC (Model–view–controller), which is a software architectural pattern for implementing user interfaces. Originating from Smalltalk, models respresent data, views represent a visual representation of the model, and controllers are the link between the user and the system. On this web page, the HTML is the model, the CSS underlying it is the view, and the browser is the controller.

ActiveRecord deals with the model, or data part, of MVC. ActiveRecord is the connection between data stored in a database to what's minipulted as part of the application. So ActiveRecord allows you to validate a state of a model, before it gets written into the database.
