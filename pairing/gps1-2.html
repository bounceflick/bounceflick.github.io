What is Pair Programming?

Pair programming is a great way for developers to divide up the work. It allows for one developer (the navigator) to direct the programming workflow while the other developer (the driver) implements the vision.

Why is pair programming important to DBC?

It allows students to learn from each other, both from listening to and explaining concepts.

Driver Role:
Has access to the keyboard.
Focuses on tactics.
Ideally has expertise in the technology being used.

Navigator Role:

  • Does not have access to the keyboard.
  • Focuses on strategies.
  • Is concerned with overall design, which tests will drive the code forward, and which refactorings will improve the entire codebase.
